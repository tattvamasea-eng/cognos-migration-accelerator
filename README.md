# Cognos → Power BI Migration Accelerator
### Proof of Concept | Life Insurance Practice

Automates IBM Cognos Analytics report migration to Microsoft Power BI. In a single `python main.py` run, it extracts, maps, scores, validates, and generates DAX measures from a Cognos XML report spec.

| | |
|---|---|
| **Stack** | Python 3.11 · React 18 · Vite · recharts · FastAPI (planned) |
| **Domain** | Life Insurance (APS, MIB, MVR, OSFI, PIPEDA, DAX) |
| **Version** | 1.0 — POC Release · May 2026 |

---

## What This Does

In a single `python main.py` run, the pipeline:

- Parses a Cognos XML report spec via XPath across 5 extraction zones
- Generates a canonical Intermediate Representation (IR) decoupled from both platforms
- Applies rule-based visual mapping (9 Cognos types → Power BI equivalents) and DAX translation (8 patterns, 150+ entries)
- Scores migration complexity (0–100, Low/Medium/High) with effort hour estimates
- Validates output against 7 QA checks (3 live, 4 stubs for production wiring)
- Outputs `migration_result.json` and a `.dax` measures file ready for Power BI Desktop

A React (Vite) demo app provides a no-backend client presentation layer with 3 tabs: **Score**, **Migrate**, and **Preview**.

## Running on macOS

**1. Clone**
```bash
git clone https://github.com/your-org/cognos-migration-accelerator.git
cd cognos-migration-accelerator
```

**2. Python Pipeline**
```bash
cd pipeline
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python3 main.py --xml mock_cognos_report.xml --output ../output/migration_result.json
```
Output files appear in `../output/`. Open `migration_result.json` to inspect the full enriched IR, complexity score, visual mappings, and warnings.

**3. React Demo**
```bash
cd ../frontend
npm install
npm run dev
```
Open browser: [http://localhost:5175](http://localhost:5175)

## Regulatory Notes

- **OSFI B-20 / E-21:** Rule engine and scorer are models under OSFI definition. Governance documentation required before production use in underwriting decisions.
- **PIPEDA:** Pipeline IR contains field names only — no PII values. Azure deployment must use Canada Central or Canada East regions.
- **RLS:** Power BI output must implement row-level security aligned to the client's data access matrix.
- **Claude API:** If AI-assisted DAX translation is enabled, confirm client AI use policy and Anthropic data processing addendum.
